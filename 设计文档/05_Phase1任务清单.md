# Phase 1 任务清单

## 1. Phase 1 目标

Phase 1 的目标是完成单细胞 RNA-seq 智能分析平台的最小可用闭环。

闭环包括：

用户上传表达矩阵  
→ 平台自动识别格式  
→ 平台检查 metadata  
→ 平台推荐分析模式  
→ 执行 QC、聚类、注释  
→ 执行细胞组成分析  
→ 根据条件执行 pseudobulk 差异分析  
→ 执行 GO/KEGG/GSEA 富集  
→ 生成图表和标准科研报告

Phase 1 不追求一次性实现所有高级模块，重点是保证基础流程稳定、结果可追溯、结论有可信度标注。

### 2.1 纳入范围

Phase 1 纳入：

1. 数据上传与格式识别。
2. Metadata 模板生成与编辑。
3. 实验设计识别。
4. 分析模式自动推荐。
5. QC。
6. 过滤。
7. 归一化。
8. PCA / UMAP。
9. 聚类。
10. marker gene 分析。
11. 细胞类型自动注释。
12. 人工复核细胞注释。
13. 细胞组成分析。
14. pseudobulk 差异分析。
15. GO/KEGG/Reactome 富集。
16. GSEA。
17. 常用图表生成。
18. 自动报告生成。
19. 软件和数据库版本记录。
20. 结果可信度标注。

### 2.2 暂不纳入范围

Phase 1 暂不纳入：

1. 项目管理、权限和审计系统。
2. 细胞通讯分析。
3. 轨迹分析。
4. inferCNV / CopyKAT。
5. SCENIC。
6. TCR/BCR。
7. 空间转录组。
8. scATAC-seq。
9. CITE-seq。
10. 多组学整合。
11. 企业级部署。
12. 临床诊断功能。

## 5. Phase 1 最终验收清单

Phase 1 最终应满足：

- [ ] 用户可以上传单细胞表达矩阵。
- [ ] 平台可以识别矩阵格式。
- [ ] 平台可以判断 raw count。
- [ ] 平台可以提示是否缺 metadata。
- [ ] 平台可以生成 metadata 模板。
- [ ] 用户可以编辑 metadata。
- [ ] 平台可以识别实验设计。
- [ ] 平台可以推荐分析模式。
- [ ] 平台可以给出结果可信度等级。
- [ ] 平台可以完成 QC。
- [ ] 平台可以完成过滤。
- [ ] 平台可以完成归一化。
- [ ] 平台可以完成 PCA 和 UMAP。
- [ ] 平台可以完成聚类。
- [ ] 平台可以输出 marker gene。
- [ ] 平台可以进行细胞类型注释。
- [ ] 用户可以人工复核注释。
- [ ] 平台可以计算细胞组成。
- [ ] 平台可以进行 pseudobulk 差异分析。
- [ ] 平台可以输出 DEG 表。
- [ ] 平台可以生成火山图。
- [ ] 平台可以进行 GO/KEGG/Reactome 富集。
- [ ] 平台可以进行 GSEA。
- [ ] 平台可以生成主要图表。
- [ ] 平台可以生成 Markdown 报告。
- [ ] 平台可以生成 PDF 报告。
- [ ] 报告包含结果可信度标注。
- [ ] 报告包含方法学说明。
- [ ] 报告包含软件版本。
- [ ] 报告包含数据库版本。
- [ ] 探索性结果不会被写成正式统计结论。
- [ ] 不满足条件的分析会被自动降级或禁止。
