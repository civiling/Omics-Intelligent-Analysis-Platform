# 前端页面结构设计文档

## 1. 设计目标

前端页面不仅用于展示图表，还应承担数据上传、格式识别、metadata 构建、分析模式推荐、分析任务配置、结果可视化、报告生成和结果解释等功能。

平台前端应帮助科研用户回答四个问题：

1. 我的数据能不能分析？
2. 我的数据适合做哪些分析？
3. 分析结果是否可靠？
4. 这些结果在生物学上如何解释？

---

## 2. 页面总体结构

第一阶段建议采用以下主菜单：

1. 首页 / 分析概览
2. 数据上传与格式识别
3. Metadata 构建与实验设计
4. 分析模式推荐
5. 数据质控
6. 细胞聚类与注释
7. 细胞组成分析
8. 差异表达分析
9. 富集与通路分析
10. 报告生成

后续扩展菜单：

1. 肿瘤微环境分析
2. 细胞通讯分析
3. 轨迹与状态转变
4. 调控网络分析
5. 多组学整合分析

## 3. 页面一：首页 / 分析概览

### 3.1 页面目标

展示当前数据集的整体状态、可执行分析、已完成分析和关键结果摘要。

### 3.2 页面模块

1. 项目名称。
2. 数据集名称。
3. 数据类型。
4. 样本数量。
5. 细胞数量。
6. 基因数量。
7. metadata 状态。
8. 推荐分析模式。
9. 当前工作流状态。
10. 已生成报告。
11. 主要警告信息。

### 3.3 状态卡片

建议设置以下卡片：

1. 数据格式：已识别 / 待确认 / 失败。
2. Metadata：完整 / 部分缺失 / 缺失。
3. QC 状态：通过 / 警告 / 失败。
4. 分析模式：描述性 / 探索性 / 正式统计 / 推断性。
5. 工作流状态：未开始 / 运行中 / 已完成 / 失败。
6. 报告状态：未生成 / 可生成 / 已生成。

### 3.4 示例提示

当前数据包含 6 个样本，其中 Normal 3 个、Tumor 3 个。已识别 sample_id 和 condition，但 patient_id 信息不完整。平台建议进行探索性 normal vs tumor pseudobulk 差异分析，并提示补充 patient_id 以支持配对设计。

## 4. 页面二：数据上传与格式识别

### 4.1 页面目标

允许用户上传表达矩阵和 metadata，并自动识别数据格式。

### 4.2 主要功能

1. 文件拖拽上传。
2. 批量上传。
3. 支持 csv.gz、tsv.gz、mtx、h5、h5ad、rds 等格式。
4. 自动识别文件类型。
5. 自动识别矩阵方向。
6. 自动识别 raw count 或 normalized expression。
7. 自动识别物种。
8. 显示文件解析结果。
9. 显示格式问题和修复建议。

### 4.3 文件列表字段

| 字段 | 说明 |
|---|---|
| 文件名 | 原始文件名 |
| 文件大小 | 文件大小 |
| 文件类型 | csv.gz / h5ad / mtx / h5 |
| 识别角色 | expression matrix / metadata / unknown |
| 解析状态 | pending / success / failed |
| 样本 ID | 自动推断或用户填写 |
| 警告 | 格式问题提示 |

### 4.4 格式识别摘要

显示：

1. 矩阵方向。
2. 基因数。
3. 细胞数。
4. 数值类型。
5. 物种。
6. gene ID 类型。
7. 是否存在重复基因名。
8. 是否存在空行或空列。
9. 是否需要转置。
10. 是否适合差异分析。

### 4.5 操作按钮

1. 上传文件。
2. 删除文件。
3. 重新解析。
4. 确认矩阵方向。
5. 自动转置。
6. 下载格式识别报告。
7. 进入 metadata 构建。

## 5. 页面三：Metadata 构建与实验设计

### 5.1 页面目标

帮助用户补充 sample-level metadata，并识别实验设计。

### 5.2 Metadata 表格字段

建议默认包含：

1. sample_id
2. file_name
3. condition
4. patient_id
5. batch
6. tissue
7. disease
8. species
9. time_point
10. dose
11. paired_status
12. metadata_source
13. confirmation_status

### 5.3 功能

1. 自动生成 metadata 草稿。
2. 从文件名识别 sample_id。
3. 从文件名识别 condition。
4. 从 GSM 编号识别公开数据库信息。
5. 用户手动编辑 metadata。
6. 批量填充 condition。
7. 批量填充 batch。
8. 批量填充 tissue。
9. 检查缺失值。
10. 检查 condition 是否足够。
11. 检查每组样本数。
12. 检查配对关系。
13. 保存 metadata。
14. 导出 metadata.csv。

### 5.4 实验设计识别区

显示：

1. 样本总数。
2. 分组数量。
3. 每组样本数。
4. 是否配对。
5. 是否有 batch。
6. 是否有 time_point。
7. 是否有 dose。
8. 是否满足正式差异分析条件。
9. 推荐分析模式。

### 5.5 提示规则

如果没有 condition：

提示：当前 metadata 缺少 condition，只能进行单样本或多样本整合分析，不能进行组间差异分析。

如果 condition 只有一组：

提示：当前数据只有一个分组，不能进行 normal vs tumor 或 control vs treatment 差异分析。

如果每组样本数不足：

提示：当前数据可进行探索性差异分析，但不建议作为正式统计结论。

如果 patient_id 完整且配对：

提示：检测到配对实验设计，平台建议使用 design = ~ patient_id + condition。

## 6. 页面四：分析模式推荐

### 6.1 页面目标

根据数据格式、metadata 和质控情况，自动推荐可执行分析模式，并显示每种模式的可信度等级。

### 6.2 页面结构

1. 数据条件摘要。
2. 可执行分析模式。
3. 不建议执行的分析。
4. 缺失信息。
5. 结果可信度说明。
6. 一键启动推荐工作流。

### 6.3 分析模式卡片

每张卡片包含：

1. 模式名称。
2. 适用条件。
3. 当前是否满足。
4. 结果等级。
5. 需要的输入。
6. 输出内容。
7. 风险提示。
8. 启动按钮。

### 6.4 分析模式示例

1. 单样本细胞组成分析：可执行。
2. 多样本整合分析：可执行。
3. 探索性 pseudobulk 差异分析：可执行，但样本数不足。
4. 正式 pseudobulk 差异分析：暂不推荐，缺少生物学重复。
5. 配对差异分析：暂不推荐，缺少 patient_id。
6. 富集与通路分析：依赖 marker 或 DEG 结果。
7. 细胞通讯分析：后续阶段支持。

### 6.5 可信度标签

1. 描述性结果。
2. 探索性结果。
3. 正式统计结果。
4. 推断性结果。

标签应在页面和报告中保持一致。

## 7. 页面五：数据质控

### 7.1 页面目标

展示 QC 指标、过滤阈值、过滤前后变化，并允许用户调整参数。

### 7.2 QC 指标

1. nCount_RNA。
2. nFeature_RNA。
3. percent.mt。
4. percent.ribo。
5. 每个样本细胞数。
6. 每个样本 UMI 分布。
7. 每个样本基因数分布。

### 7.3 图表

1. nFeature violin plot。
2. nCount violin plot。
3. percent.mt violin plot。
4. nCount vs nFeature scatter plot。
5. percent.mt vs nCount scatter plot。
6. 过滤前后细胞数柱状图。
7. 样本级 QC summary table。

### 7.4 参数设置

1. min_features。
2. max_features。
3. min_counts。
4. max_counts。
5. max_percent_mt。
6. min_cells_per_gene。
7. 是否按样本单独过滤。
8. 是否保留低质量细胞用于对比查看。

### 7.5 输出

1. QCResult。
2. FilteredMatrix。
3. QC 图表。
4. QC 结论。
5. 过滤日志。

## 8. 页面六：细胞聚类与注释

### 8.1 页面目标

展示聚类、降维、marker gene 和细胞类型注释结果，支持人工复核。

### 8.2 页面模块

1. UMAP / tSNE 聚类图。
2. 按 sample 分组 UMAP。
3. 按 condition 分组 UMAP。
4. 按 cell type 分组 UMAP。
5. cluster marker 表。
6. cell type 注释表。
7. marker gene dot plot。
8. feature plot。
9. 细胞注释人工复核表。

### 8.3 聚类参数

1. PCA 维度。
2. clustering resolution。
3. neighbor 数量。
4. UMAP 参数。
5. 是否重新聚类。

### 8.4 注释结果表字段

| 字段 | 说明 |
|---|---|
| cluster_id | cluster 编号 |
| predicted_cell_type | 自动预测细胞类型 |
| final_cell_type | 最终细胞类型 |
| confidence_score | 置信度 |
| supporting_markers | 支持 marker |
| negative_markers | 阴性 marker |
| method | 注释方法 |
| review_status | auto / reviewed / modified |

### 8.5 人工复核功能

1. 修改细胞类型名称。
2. 合并 cluster。
3. 标记低置信度 cluster。
4. 上传自定义 marker。
5. 保存最终注释。
6. 重新生成细胞组成分析。
7. 重新触发 pseudobulk 分析。

## 9. 页面七：细胞组成分析

### 9.1 页面目标

展示每个样本、每个分组、每个 condition 下的细胞组成。

### 9.2 图表

1. 每个样本细胞类型堆叠柱状图。
2. 每个 condition 细胞类型比例图。
3. cell type × sample 热图。
4. cell type × condition 热图。
5. UMAP 上按 cell type 显示。
6. UMAP 上按 sample 显示。

### 9.3 表格

1. 每个样本中每类细胞数量。
2. 每个样本中每类细胞比例。
3. 每个 condition 中每类细胞平均比例。
4. 细胞比例组间比较结果。

### 9.4 结果等级

1. 单样本：描述性结果。
2. 多样本无重复：探索性结果。
3. 多样本有重复：可进行正式比例差异统计。

## 10. 页面八：差异表达分析

### 10.1 页面目标

展示 DEG 结果，区分探索性差异分析和正式 pseudobulk 差异分析。

### 10.2 页面模块

1. 差异分析设计摘要。
2. contrast 选择。
3. cell type 选择。
4. DEG summary。
5. volcano plot。
6. MA plot。
7. heatmap。
8. DEG table。
9. 上调基因表。
10. 下调基因表。
11. 下载结果。

### 10.3 差异分析设计摘要

显示：

1. 分析模式。
2. contrast。
3. 使用方法。
4. design formula。
5. 样本数。
6. 每组样本数。
7. 是否配对。
8. 是否纳入 batch。
9. 结果可信度等级。

### 10.4 DEG 表字段

1. gene
2. baseMean
3. log2FoldChange
4. lfcSE
5. stat
6. pvalue
7. padj
8. regulation
9. cell_type
10. contrast

### 10.5 提示语

如果为探索性分析：

当前差异分析基于有限生物学重复，结果仅用于候选基因筛选，不建议作为正式统计结论。

如果为正式分析：

当前分析基于 sample-level pseudobulk，并以样本作为生物学重复，适合进行正式差异表达分析。

### 11.3 图表

2. KEGG bubble plot。
3. Reactome bar plot。
4. GSEA enrichment plot。
5. pathway heatmap。
6. dot plot。
7. ridge plot 可选。

### 11.4 表格

1. pathway_id
2. pathway_name
3. gene_count
4. gene_ratio
5. pvalue
6. p.adjust

## 13. 交互设计原则

1. 用户不需要一开始选择复杂工具。
2. 平台先判断数据条件，再推荐分析。
3. 不满足条件的分析应显示为不可用或降级。
4. 所有正式统计结果必须显示样本数和设计公式。
5. 所有探索性结果必须显示风险提示。
6. 所有推断性结果必须标明“基于表达推断”。
7. 图表应支持导出。
8. 表格应支持搜索、排序、筛选和下载。
9. 报告应支持重新生成。
10. 用户修改细胞注释后，应提示是否重新运行后续分析。
