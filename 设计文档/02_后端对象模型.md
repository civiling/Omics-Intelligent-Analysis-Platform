# 后端对象模型设计文档

## 1. 设计目标

后端对象模型用于统一平台内部数据结构，使不同分析模块之间可以稳定流转。平台不能让每个工具各自输出一套无法复用的文件格式，而应将表达矩阵、metadata、QC 结果、聚类结果、差异分析结果、富集结果、报告对象等抽象为标准对象。

标准对象设计目标：

1. 降低模块之间耦合。
2. 支持工作流节点复用。
3. 支持结果追溯。
4. 支持图表和报告统一调用。
5. 支持后续扩展至空间转录组、scATAC、多组学等场景。

## 3. Project 对象

### 3.1 用途

Project 表示一个分析项目。第一阶段虽然暂不建设复杂项目管理和权限系统，但后端仍建议保留 Project 对象，以便管理一次完整分析。

### 3.2 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| project_id | string | 项目唯一 ID |
| name | string | 项目名称 |
| description | string | 项目描述 |
| organism | string | 物种，如 human、mouse |
| disease_context | string | 疾病背景，如 gastric cancer |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |
| status | string | active / archived |
| default_analysis_mode | string | 自动推荐的默认分析模式 |

### 3.3 示例

{
  "project_id": "proj_001",
  "name": "GSE183904 gastric cancer scRNA analysis",
  "organism": "human",
  "disease_context": "gastric cancer",
  "status": "active"
}

## 6. ExpressionMatrix 对象

### 6.1 用途

ExpressionMatrix 表示表达矩阵，是平台最核心的数据对象之一。

### 6.2 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| matrix_id | string | 表达矩阵 ID |
| dataset_id | string | 所属数据集 |
| file_id | string | 来源文件 |
| matrix_type | string | raw_count / normalized / scaled |
| orientation | string | gene_by_cell / cell_by_gene |
| organism | string | human / mouse / unknown |
| gene_id_type | string | symbol / ensembl / mixed |
| n_genes | integer | 基因数 |
| n_cells | integer | 细胞数 |
| sparse_format | string | csr / csc / dense |
| storage_path | string | 标准化后矩阵路径 |
| is_raw_count | boolean | 是否为 raw count |
| validation_status | string | valid / warning / invalid |
| warnings | array | 格式警告 |

### 6.3 关键规则

1. 差异分析优先使用 raw_count 矩阵。
2. 如果检测为 normalized matrix，应提示不能直接用于 DESeq2 原始 count 差异分析。
3. 如果矩阵方向错误，应自动转置或提示用户确认。
4. 重复基因名应在解析阶段合并或重命名，并记录处理方式。

## 14. PseudobulkMatrix 对象

### 14.1 用途

保存 sample × cell type 层面的 pseudobulk count，是正式差异分析的核心输入。

### 14.2 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| pseudobulk_id | string | pseudobulk ID |
| matrix_id | string | 原始表达矩阵 |
| annotation_id | string | 细胞类型注释 |
| sample_metadata_id | string | 样本 metadata |
| group_by | string | sample_id + cell_type |
| aggregation_method | string | sum_counts |
| min_cells_per_group | integer | 每组最少细胞数 |
| pseudobulk_matrix_path | string | pseudobulk 矩阵路径 |
| sample_info_path | string | pseudobulk 样本信息 |
| excluded_groups | array | 被排除的组 |
| created_at | datetime | 创建时间 |

### 14.3 关键规则

1. pseudobulk 应使用 raw count 聚合。
2. 聚合单位应为 sample_id × cell_type。
3. 低细胞数的 sample-cell_type 组合应标记或排除。
4. 差异分析不能把单个细胞当作生物学重复。

## 27. 对象设计原则

1. 所有对象必须有唯一 ID。
2. 所有对象必须记录来源对象。
3. 所有结果对象必须记录生成参数。
4. 所有分析结果必须可以追溯到输入文件。
5. 所有统计结果必须记录软件版本。
6. 所有数据库依赖结果必须记录数据库版本。
7. 所有可视化图表必须关联来源结果。
8. 所有报告结论必须关联证据对象或结果对象。
9. 所有结果对象必须包含可信度等级。
10. 所有对象应支持后续扩展字段。
